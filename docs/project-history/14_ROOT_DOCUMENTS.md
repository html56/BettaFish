# 根目录文档说明

本文件说明项目根目录中保留的重要 Markdown 文档。

## 📄 根目录文档列表

以下文档位于项目根目录 (`/`),不在 `docs/project-history/` 目录中:

### 1. README.md
- **位置**: `/README.md`
- **说明**: 项目主 README 文件 (中文版)
- **内容**:
  - 项目简介
  - 功能特性
  - 安装部署
  - 使用说明
  - 技术架构
  - 贡献指南

### 2. README-EN.md
- **位置**: `/README-EN.md`
- **说明**: 项目主 README 文件 (英文版)
- **内容**: 与中文版 README 相同,提供英文版本

## 📂 文档组织原则

### 保留在根目录的文档
- **README.md / README-EN.md**: 项目主文档,GitHub 默认显示
- **LICENSE**: 开源协议文件
- **.gitignore**: Git 忽略规则

### 整理到 docs/ 的文档
- **历史文档**: 部署、设计、优化等过程文档
- **技术文档**: API、架构、开发规范等
- **用户文档**: 使用指南、配置说明等

## 🔗 快速访问

### 根目录文档
- [README.md](../../README.md) - 项目主文档 (中文)
- [README-EN.md](../../README-EN.md) - 项目主文档 (英文)

### docs 目录文档
- [docs/README.md](../README.md) - 文档中心主页
- [docs/project-history/](./README.md) - 项目历史文档索引

## 📝 说明

根目录的 README 文件是项目的门面,应始终保留在根目录,以便:
1. GitHub 自动显示项目说明
2. 用户克隆仓库后首先看到
3. 符合开源项目的标准结构

历史文档和技术文档则整理到 `docs/` 目录,保持根目录的简洁。
